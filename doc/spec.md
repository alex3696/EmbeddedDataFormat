EDF (Embedded Data Format) — Краткая спецификация

1. Основные принципы
Блочный формат для встраиваемых систем (≤ 1024 байт RAM)
Без malloc — линейный аллокатор в статическом буфере
Append-only — подходит для флэш-памяти
Поддержка CRC16 (в текстовом режиме CRC не используется)
Бинарное и текстовое представление (конвертируемы)
Little-endian (т.к. современных big-endian не существует), UTF-8
Макс. длина строки-	255 байт (обрезается)
размер блока	256-4096 байт

2. Структура блока
text
┌────────┬────────┬──────────┬────────────┬───────┐
│ Type   │ Seq    │ Len      │ Data       │ CRC   │
│ 1 байт │ 1 байт │ 2 байта  │ Len байт   │ 2 байта│
└────────┴────────┴──────────┴────────────┴───────┘
Поле	Описание
Type	Тип блока
Seq	Порядковый номер (инкрементируется с переполнением)
Len	Длина поля Data
Data	Данные блока
CRC	CRC16

3. Типы блоков
Тип	Значение	Символ	Текстовый вид
Заголовок	126 (0x7E)	~	<~ ... >
Схема	63 (0x3F)	?	<? ... >
Данные	61 (0x3D)	=	<= ... >

4. Заголовок файла

typedef struct EdfHeader {
    uint8_t VersMajor;      // 1
    uint8_t VersMinor;      // 0
    uint16_t Encoding;      // 65001 = UTF-8
    uint16_t Blocksize;     // 256-4096
    uint32_t Flags;          // пока не задействовано
} EdfHeader_t;
в бинарном формате занимает 16 байт данных, дополняется нулями

Текстовый вид:

<~ {version=1.0; bs=512; encoding=65001; flags=0; } >

5. Схема данных

Примитивные типы
Тип	Код	Размер	Тип	Код	Размер
Struct	0	-	UInt32	7	4
Char	1	1	Int64	8	8
Int8	2	1	UInt64	9	8
UInt8	3	1	Half	10	2
Int16	4	2	Single	11	4
UInt16	5	2	Double	12	8
Int32	6	4	String	13	var

Char - специальный тип, для записи ansi строк фиксированной размерности char[10] SomesTring; Пустые элементы в конце дополняются '\0'.


Бинарный формат схемы TypeRec
┌──────────────┬─────────────────────────┐
│ Id (4 байта) │ Идентификатор переменной │
├──────────────┼─────────────────────────┤
│ TypeInfo     │ Рекурсивное описание     │
├──────────────┼─────────────────────────┤
│ Name         │ Имя переменной (строка)  │
├──────────────┼─────────────────────────┤
│ Desc         │ Описание (опционально)   │
└──────────────┴─────────────────────────┘

где TypeInfo:
┌───────────┬─────────────────────────────┐
│ Type      │ Примитивный или составной тип (1 байт)    │
├───────────┼─────────────────────────────┤
│ DimsCount │ Количество измерений (1байт)│
├───────────┼─────────────────────────────┤
│ Dims[*]   │ Размеры (uint32_t)          │
├───────────┼─────────────────────────────┤
│ Name      │ Имя типа (строка)           │
├───────────┼─────────────────────────────┤
│ ChildsCnt │ Кол-во дочерних (Struct) (1 байт)    │
├───────────┼─────────────────────────────┤
│ Childs[*] │ Элементы структуры          │
└───────────┴─────────────────────────────┘

Текстовый вид  где значения в [] необязательные
<? {TypeRec.Id, TypeRec.Name[, TypeRec.Desc]}
   TypeInfo.Type [TypeInfo.Name] [ [TypeInfo.Dims.Item[0]...[TypeInfo.Dims.Item[TypeInfo.Dims.Count]]
   [{ тут поля структуры рекурсивно, если тип Struct }]
   >

<? {0;"VariableKV"} Struct "KeyValue" {
  String "Key";
  String "Value";
} >

<? {0;"weight variable"} Int32; >

<? {0;"Char Text"} Char[20]; >

6. Данные
Текстовый вид
<= {"Key1";"Value1";}>
<= -1;>
<= 1.1;>
<= "String Value";>

Бинарный формат
Тип	Бинарный формат
Числа (1-8 байт)	LE, прямой memcpy
String	[Len:1][Data:Len] (Len ≤ 255)
Struct	Поля последовательно
Массивы	Элементы последовательно

Важные правила
Правило	Описание
Примитив	Не может быть разорван между блоками
Элемент массива	Может переходить в следующий блок
Строка	Максимум 255 байт (обрезается)

7. Пример текстового файла

<~ {version=1.0; bs=512; encoding=65001; flags=0; } >

<? {0;"SensorData"} Struct {
  UInt32 "timestamp";
  Single "value";
  String "unit";
} >

<= {1700000000; 23.5;"°C";}>
<= {1700000060; 23.6;"°C";}>

<? {0;"Pressure"} Struct {
  UInt32 "timestamp";
  Single "value";
  String "unit";
} >

<= {1700000000; 1013.2;"hPa";}>
<= {1700000060; 1013.5;"hPa";}>

8. API
c
// Открытие/закрытие
int EdfOpen(EdfWriter_t* edf, const char* file, const char* mode);
int EdfClose(EdfWriter_t* dw);

// Запись
int EdfWriteHeader(EdfWriter_t* dw, const EdfHeader_t* h, size_t* writed);
int EdfWriteInfo(EdfWriter_t* dw, const TypeRec_t* t, size_t* writed);
int EdfWriteDataBlock(EdfWriter_t* dw, const void* src, size_t srcLen);
int EdfFlushDataBlock(EdfWriter_t* dw, size_t* writed);
Режимы: "wb"/"rb" — бинарные, "wt"/"rt" — текстовые, "ab"/"at" — дозапись
