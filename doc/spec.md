# EDF (Embedded Data Format) — Спецификация

## 1. Основные принципы
- Блочный формат для встраиваемых систем (RAM ≤ 1024 байт)
- Без `malloc` — линейный аллокатор в статическом буфере
- Append-only — подходит для флэш-памяти
- Поддержка CRC16 (в текстовом режиме CRC не используется)
- Бинарное и текстовое представление (конвертируемы)
- Little-endian, UTF-8
- Макс. длина строки: 255 байт (обрезается без предупреждения)
- Размер блока: 256-4096 байт (задаётся в заголовке)

## 2. Структура блока
┌────────┬────────┬──────────┬────────────┬───────┐
│ Type │ Seq │ Len │ Data │ CRC │
│ 1 байт │ 1 байт │ 2 байта │ Len байт │ 2 байта│
└────────┴────────┴──────────┴────────────┴───────┘
| Поле | Описание |
|------|----------|
| Type | Тип блока (см. раздел 3) |
| Seq  | Порядковый номер (инкрементируется, переполнение разрешено) |
| Len  | Длина поля Data (0 ≤ Len ≤ BlockSize) |
| Data | Данные блока |
| CRC  | CRC16 (Modbus), вычисляется от Type + Seq + Len + Data |

**Примечание**: В текстовом режиме поле CRC отсутствует, блок завершается символом `>`.

## 3. Типы блоков

| Тип | Значение | Символ | Текстовый вид | Назначение |
|-----|----------|--------|---------------|------------|
| Заголовок | 126 (0x7E) | `~` | `<~ ... >` | Заголовок файла |
| Схема | 63 (0x3F) | `?` | `<? ... >` | Описание типа данных |
| Данные | 61 (0x3D) | `=` | `<= ... >` | Данные переменной |

## 4. Заголовок файла

```c
typedef struct EdfHeader {
    uint8_t VersMajor;      // 1
    uint8_t VersMinor;      // 0
    uint16_t Encoding;      // 65001 = UTF-8
    uint16_t Blocksize;     // размер блока (256-4096)
    uint32_t Flags;         // зарезервировано, должен быть 0
} EdfHeader_t;
```
В бинарном формате занимает 16 байт, остальные байты блока (до Blocksize) заполняются нулями
Заголовок всегда находится в первом блоке файла
Текстовый вид:
<~ {version=1.0; bs=512; encoding=65001; flags=0; } >

## 5. Схема данных

## 5.1 Примитивные типы
Тип	Код	Размер (байт)	C-тип	Примечание
Struct	0	-	-	Составной тип
Char	1	1	char[]	Фиксированный массив ANSI-строк
Int8	2	1	int8_t	
UInt8	3	1	uint8_t	
Int16	4	2	int16_t	
UInt16	5	2	uint16_t	
Int32	6	4	int32_t	
UInt32	7	4	uint32_t	
Int64	8	8	int64_t	
UInt64	9	8	uint64_t	
Half	10	2	-	16-битный float (IEEE 754-2008)
Single	11	4	float	
Double	12	8	double	
String	13	var	char*	Динамическая строка (см. раздел 6)
Особенности типов:

- Char — используется для строк фиксированной длины (например, char[10]). В бинарном виде хранится как сырые байты. В текстовом виде заключается в кавычки.
Пустые элементы в конце дополняются '\0'.
- String — динамическая строка, в бинарном виде имеет префикс длины (1 байт).


## 5.2 Бинарный формат TypeRec
┌──────────────┬─────────────────────────┐
│ Id (4 байта) │ Идентификатор переменной │
├──────────────┼─────────────────────────┤
│ TypeInfo     │ Рекурсивное описание     │
├──────────────┼─────────────────────────┤
│ Name         │ Имя переменной (строка)  │
├──────────────┼─────────────────────────┤
│ Desc         │ Описание (опционально)   │
└──────────────┴─────────────────────────┘

## 5.3 Бинарный формат TypeInfo
┌───────────┬─────────────────────────────┐
│ Type      │ Примитивный или составной тип (1 байт)    │
├───────────┼─────────────────────────────┤
│ DimsCount │ Количество измерений (1байт(0-255))│
├───────────┼─────────────────────────────┤
│ Dims[*]   │ Размеры (uint32_t)          │
├───────────┼─────────────────────────────┤
│ Name      │ Имя типа (строка)           │
├───────────┼─────────────────────────────┤
│ ChildsCnt │ Кол-во дочерних (Struct) (1 байт)    │
├───────────┼─────────────────────────────┤
│ Childs[*] │ Элементы структуры(TypeInfo)│
└───────────┴─────────────────────────────┘
Примечания:
- ChildsCount присутствует только если Type == Struct и ChildsCount > 0
- Поля структуры располагаются последовательно в том же порядке, что и в Childs

## 5.4 Текстовый формат схемы
где значения в [] необязательные
<? {TypeRec.Id, TypeRec.Name[, TypeRec.Desc]}
   TypeInfo.Type [TypeInfo.Name] [ [TypeInfo.Dims.Item[0]...[TypeInfo.Dims.Item[TypeInfo.Dims.Count]]
   [{ тут поля структуры рекурсивно, если тип Struct }]
   >
Примеры:
```
// Простая переменная
<? {0;"weight variable"} Int32; >

// Массив Char[20]
<? {0;"Char Text"} Char[20]; >

// Структура
<? {0;"SensorData"} Struct "Sensor" {
    UInt32 "timestamp";
    Single "value";
    String "unit";
} >

// Вложенная структура с массивом
<? {0;"Complex"} Struct {
    Int64 "time";
    Struct "State"[3] {
        Int8 "text";
        Struct "Pos" {
            Int32 "x";
            Int32 "y";
        };
        Double "Temp"[2][2];
    };
} >
```

## 6. Формат данных

## 6.1 Бинарный формат
Тип	Бинарный формат	Примечание
Числа (1-8 байт)	LE, прямой memcpy	
String	[Len:1][Data:Len]	Len ≤ 255, Data может не содержать \0
Char	сырые байты	Фиксированная длина из Dims
Struct	поля последовательно	В порядке объявления
Массивы	элементы последовательно	Row-major order

Важные правила:
- Примитив не может быть разорван между блоками
- Элемент массива может переходить в следующий блок
- Строки обрезаются до 255 байт (без предупреждения)

## 6.2 Текстовый формат
```
<= {"Key1";"Value1";}>
<= -1;>
<= 1.1;>
<= "String Value";>
```
Правила сериализации в текст:
- Числа: десятичное представление
- Строки: в двойных кавычках, экранирование не поддерживается
- Структуры: значения перечисляются через ; внутри {}
- Массивы: как структуры
- Char массивы: как строки в кавычках

## 6.3  Пример текстового файла
```
<~ {version=1.0; bs=512; encoding=65001; flags=0; } >

<? {0;"SensorData"} Struct {
  UInt32 "timestamp";
  Single "value";
  String "unit";
} >

<= {1700000000; 23.5;"°C";}>
<= {1700000060; 23.6;"°C";}>

<? {0;"Pressure"} Struct {
  UInt32 "timestamp";
  Single "value";
  String "unit";
} >

<= {1700000000; 1013.2;"hPa";}>
<= {1700000060; 1013.5;"hPa";}>
```

## 7. Чтение множественных схем
При чтении EDF файла схема десериализуется во внутреннее представление с использованием линейного аллокатора.
При чтении EDF файла с несколькими блоками `btVarInfo` необходимо:

1. Сбрасывать позицию записи в буфере (`mem.WPos = 0`)
2. Обнулять указатель на схему (`br.t = NULL`)
3. Использовать один и тот же буфер для последовательного чтения

Пример корректной обработки:

```c
MemStream_t msDst = {0};
MemStreamOutOpen(&msDst, buffer, buffer_size);

while (!EdfReadBlock(&br)) {
    if (br.BlkType == btVarInfo) {
        msDst.WPos = 0;     // ← сброс позиции
        br.t = NULL;        // ← сброс указателя
        StreamWriteBinToCBin(br.Block, br.DatLen, NULL,
                              buffer, buffer_size, NULL, &br.t);
        // Обработка схемы...
    }
}
```
Рекомендации:
- Файл должен содержать не более одной схемы (TypeRec) на сессию чтения
- При необходимости чтения нескольких схем используйте временный буфер
- После завершения работы с файлом буфер освобождается при вызове EdfClose

## 8. API Reference
```c
// Открытие/закрытие
// Режимы: "wb"/"rb" — бинарные, "wt"/"rt" — текстовые, "ab"/"at" — дозапись
int EdfOpen(EdfWriter_t* edf, const char* file, const char* mode);
int EdfClose(EdfWriter_t* dw);

// Запись
int EdfWriteHeader(EdfWriter_t* dw, const EdfHeader_t* h, size_t* writed);
int EdfWriteInfo(EdfWriter_t* dw, const TypeRec_t* t, size_t* writed);
int EdfWriteDataBlock(EdfWriter_t* dw, const void* src, size_t srcLen);
int EdfFlushDataBlock(EdfWriter_t* dw, size_t* writed);

// Чтение
int EdfReadBlock(EdfWriter_t* dr);
int EdfReadBin(const TypeInfo_t* t, MemStream_t* src, MemStream_t* mem, 
               void** presult, size_t* resultPrimOffset, size_t* primReaded);

// Утилиты
int EdfWriteInfData(EdfWriter_t* dw, uint32_t id, PoType pt, 
                    char* name, const void* data);
```
