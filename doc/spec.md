# EDF (Embedded Data Format) — Спецификация
это бинарно-текстовый формат данных, разработанный для встраиваемых систем 
с ограниченными ресурсами (RAM ≤ 1024 байт). Формат обеспечивает:
Хранение структурированных данных во флэш-памяти
Обмен данными между embedded-устройствами и ПК
Конвертацию между бинарным и текстовым представлением

## 1. Основные принципы
- Самодостаточность - Схема данных хранится вместе с данными
- Простота — не требует сторонних библиотек
- Расширяемость — пользовательские структуры и типы
- Блочный формат для встраиваемых систем (RAM ≤ 1024 байт)
- Без `malloc` — линейный аллокатор в статическом буфере
- Append-only — подходит для флэш-памяти, потоковой записи для телеметрии и логов
- Поддержка CRC16 (в текстовом режиме CRC не используется)
- Бинарное и текстовое представление (конвертируемы)
- Little-endian, UTF-8
- Макс. длина строки: 255 байт (обрезается без предупреждения)
- Размер блока: 256-4096 байт (задаётся в заголовке)

## 2. Структура блока (EdfBlock_t)
| Поле | Размер | Описание |
|------|------  |----------|
| Type | 1 | Тип блока (см. раздел 3) |
| Seq  | 1 | Порядковый номер (инкрементируется, переполнение разрешено) |
| Len  | 2 | Длина поля Data (0 ≤ Len ≤ BlockSize) |
| Data | Len | Данные блока |
| CRC  | 2 | CRC16 (Modbus), вычисляется от Type + Seq + Len + Data |

**Примечание**: В текстовом режиме поле CRC отсутствует, блок завершается символом `>`.

## 3. Типы блоков (EdfBlockType)

| Тип | Значение | Символ | Текстовый вид | Назначение |
|-----|----------|--------|---------------|------------|
| Конфиг | 126 (0x7E) | `~` | `<~ ... >` | Конфиг файла |
| Схема | 63 (0x3F) | `?` | `<? ... >` | Описание типа данных |
| Данные | 61 (0x3D) | `=` | `<= ... >` | Данные переменной |

## 4. Конфиг файла

```c
typedef struct EdfConfig_t {
    uint8_t VersMajor;      // 1
    uint8_t VersMinor;      // 0
    uint16_t Encoding;      // 65001 = UTF-8
    uint16_t Blocksize;     // размер блока (256-4096)
    uint32_t Flags;         // зарезервировано, должен быть 0
} EdfHeader_t;
```
В бинарном формате занимает 16 байт, остальные байты блока (до Blocksize) заполняются нулями
Конфиг всегда находится в первом блоке файла

## 5. Схема данных

## 5.1 Примитивные типы
| Тип | Код | Размер (байт) | C-тип | Примечание
|-------|---|--------|---------------|------------|
|Struct |0  |	-	 | -             |Составной тип
|Char	|1	| 1      | char[]	     | Фиксированный массив ANSI-строк
|Int8	|2	| 1      | 	int8_t	     |
|UInt8	|3	| 1      | 	uint8_t	     |
|Int16	|4	| 2      | 	int16_t	     |
|UInt16	|5	| 2      | 	uint16_t     |	
|Int32	|6	| 4      | 	int32_t	     |
|UInt32	|7	| 4      | 	uint32_t     |	
|Int64	|8	| 8      | 	int64_t	     |
|UInt64	|9	| 8      | 	uint64_t     |	
|Half	|10	| 2      | 	16-битный float (IEEE 754-2008) |
|Single	|11	| 4      | 	float	     |
|Double	|12	| 8	     |  double	     |
|String	|13	| var    | 	char*	     | Динамическая строка (см. раздел 6) |
Особенности типов:

- Char — используется для строк фиксированной длины (например, char[10]). В бинарном виде хранится как сырые байты. В текстовом виде заключается в кавычки.
Пустые элементы в конце дополняются '\0'.
- String — динамическая строка, в бинарном виде имеет префикс длины (1 байт).

## 5.2 Бинарный формат EdfInf_t

| Поле | Размер | Описание |
|------|--------|----------|
| Id | 4 | Идентификатор переменной (0-65535) |
| Name | переменный | Имя переменной (String, см. раздел 6) |
| Desc | переменный | Описание переменной (String, опционально) |
| EdfType_t | переменный | Рекурсивное описание типа (см. раздел 5.3) |

## 5.3 Бинарный формат EdfType_t

| Поле | Размер | Описание |
|------|--------|----------|
| Type | 1 | Примитивный или составной тип (см. раздел 5.1) |
| DimsCount | 1 | Количество измерений (0-255) |
| Dims | DimsCount × 4 | Размеры каждого измерения (uint32_t, little-endian) |
| Name | переменный | Имя типа (String, см. раздел 6) |
| ChildsCount | 1 | Количество дочерних полей (только если Type == Struct) |
| Childs | переменный | Элементы структуры (массив EdfType_t, только если ChildsCount > 0) |

Примечания:
- ChildsCount присутствует только если Type == Struct и ChildsCount > 0
- Поля структуры располагаются последовательно в том же порядке, что и в Childs

## 5.4 Текстовый формат схемы
## 5.4 Конфиг файла
Текстовый вид:
```
<~ {version=1.0; bs=512; encoding=65001; flags=0; } >
```

где значения в [] необязательные
```
<? {EdfInf.Id, EdfInf.Name[, EdfInf.Desc]}
   EdfType_t.Type [EdfType_t.Name] [ [EdfType_t.Dims.Item[0]...[EdfType_t.Dims.Item[EdfType_t.Dims.Count]]
   [{ тут поля структуры рекурсивно, если тип Struct }]
   >
```
Примеры:
```
// Простая переменная
<? {0;"weight variable"} Int32; >

// Массив Char[20]
<? {0;"Char Text"} Char[20]; >

// Структура
<? {0;"SensorData"} Struct "Sensor" {
    UInt32 "timestamp";
    Single "value";
    String "unit";
} >

// Вложенная структура с массивом
<? {0;"Complex"} Struct {
    Int64 "time";
    Struct "State"[3] {
        Int8 "text";
        Struct "Pos" {
            Int32 "x";
            Int32 "y";
        };
        Double "Temp"[2][2];
    };
} >
```

## 6. Формат данных

## 6.1 Бинарный формат
| Тип | Формат | Размер | Примечание |
|-----|--------|--------|------------|
| UInt8, Int8, Char | прямая запись | 1 | Char — фиксированный массив ANSI-строк |
| UInt16, Int16, Half | прямая запись, LE | 2 | Half — 16-битный float (IEEE 754-2008) |
| UInt32, Int32, Single | прямая запись, LE | 4 | |
| UInt64, Int64, Double | прямая запись, LE | 8 | |
| String | Len (1 байт) + Data (Len байт) | переменный | Len ≤ 255, Data может не содержать '\0' |
| Struct | поля последовательно | сумма размеров полей | В порядке объявления в Childs |
| Массивы | элементы последовательно | элемент × totalElements | Row-major order |

Правила сериализации:
- Примитив не может быть разорван между блоками
- Элемент массива может переходить в следующий блок
- Строки обрезаются до 255 байт (без предупреждения)

## 6.2 Текстовый формат
| Конструкция | Формат | Пример |
|-------------|--------|--------|
| Запись | `<= { значения } >` | `<= {1700000000; 23.5;"°C"} >` |
| Число | десятичное представление | `-123`, `456`, `3.14159` |
| Строка (String) | в двойных кавычках | `"Hello World"` |
| Char-массив | в двойных кавычках | `"Fixed string"` |
| Структура | значения через `;` внутри `{}` | `{10;"text";3.14}` |
| Массив | как структура | `{1;2;3;4;5}` |
| Пустое значение | пустая строка или пропуск | `{}` или `{;;}` |
```
<= {"Key1";"Value1";}>
<= -1;>
<= 1.1;>
<= "String Value";>
```
Правила сериализации в текст:
- Числа: десятичное представление
- Строки: в двойных кавычках, экранирование не поддерживается
- Структуры: значения перечисляются через ; внутри {}
- Массивы: как структуры
- Char массивы: как строки в кавычках

## 6.3  Пример текстового файла
```
<~ {version=1.0; bs=512; encoding=65001; flags=0; } >

<? {0;"SensorData"} Struct {
  UInt32 "timestamp";
  Single "value";
  String "unit";
} >

<= {1700000000; 23.5;"°C";}>
<= {1700000060; 23.6;"°C";}>

<? {0;"Pressure"} Struct {
  UInt32 "timestamp";
  Single "value";
  String "unit";
} >

<= {1700000000; 1013.2;"hPa";}>
<= {1700000060; 1013.5;"hPa";}>
```

## 7. Чтение множественных схем
При чтении EDF файла схема десериализуется во внутреннее представление с использованием линейного аллокатора.
При чтении EDF файла с несколькими блоками `btInf` необходимо:

1. Сбрасывать позицию записи в буфере (`mem.WPos = 0`)
2. Обнулять указатель на схему (`br.t = NULL`)
3. Использовать один и тот же буфер для последовательного чтения

Пример корректной обработки:

```c
MemStream_t msDst = {0};
MemStreamOutOpen(&msDst, buffer, buffer_size);

while (!EdfReadBlock(&br)) {
    if (br.Blk.Type == btInf) {
        msDst.WPos = 0;     // ← сброс позиции
        br.t = NULL;        // ← сброс указателя
        StreamWriteBinToCBin(br.Block, br.DatLen, NULL,
                              buffer, buffer_size, NULL, &br.t);
        // Обработка схемы...
    }
}
```


## 8. API Reference
```c
// Открытие/закрытие
// Режимы: "wb"/"rb" — бинарные, "wt"/"rt" — текстовые, "ab"/"at" — дозапись

int EdfClose(EdfWriter_t* dw);

// Запись
int EdfWriteConfig(EdfWriter_t* dw, const EdfConfig_t* h, size_t* writed);
int EdfWriteInf(EdfWriter_t* dw, const EdfInf_t* t, size_t* writed);
int EdfWriteData(EdfWriter_t* dw, const void* src, size_t srcLen);
int EdfFlushData(EdfWriter_t* dw, size_t* writed);

// Чтение
int EdfReadBlock(EdfWriter_t* dr);
int EdfReadBin(const TypeInfo_t* t, MemStream_t* src, MemStream_t* mem, 
               void** presult, size_t* resultPrimOffset, size_t* primReaded);

// Утилиты
//shortcut: запись схемы + данных
int EdfWriteInfData(EdfWriter_t* dw, const EdfInf_t* ir, const void* d, size_t len);
int EdfWritePrimitiveInfData(EdfWriter_t* dw, PoType pt, uint32_t id, char* name, char* desc, const void* d);
```
